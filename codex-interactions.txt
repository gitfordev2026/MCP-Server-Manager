Design/Config Issues and Mis‑planning (ordered by impact)

Auth is token‑only at /mcp/apps, but access policies are not user‑aware. JWTAuthASGIMiddleware only validates the token; access policy checks do not use user identity or groups, and allowed_users/allowed_groups are ignored. This means any authenticated user can call any tool allowed for the owner, regardless of user‑level policy intent.
Files: backend/app/main.py, backend/app/routers/access_policies.py

Registry‑only contract is violated in multiple places.
CombinedAppsOpenAPIMCP.list_tools() always does live discovery rather than registry‑only. The catalog endpoint defaults force_refresh=True even when registry_only=True, so it still pulls upstream and mutates DB. This conflicts with the schema contract’s “DB is the source of truth”.
Files: backend/app/main.py, backend/app/routers/catalog.py, SCHEMA_CONTRACT.md

Default policy behavior is inconsistent at the schema level.
AccessPolicyModel.mode defaults to deny in the table model, but policy_utils inserts allow for default and per‑tool rows. Existing DB rows (or new rows created outside policy_utils) can be deny unintentionally. This is likely why you’re seeing mis‑planning around DB tables.
Files: backend/app/models/db_models.py, backend/app/services/policy_utils.py

Access control on admin operations is incomplete.
Server/base‑URL registration and deletion routes are not guarded by RBAC permissions. Even with AUTH_ENABLED=true, any authenticated user can register or modify.
Files: backend/app/routers/servers.py, backend/app/routers/base_urls.py

Exposure filtering is inconsistent between registry‑based catalog and runtime MCP.
exposure_service.resolve_exposable_tools filters from registry tables and respects access_mode, but runtime MCP uses live discovery, so you can see different tools in UI vs actual MCP output.
Files: backend/app/services/registry/exposure_service.py, backend/app/main.py

Documentation is out of sync with repo structure.
README references frontend/mcp-dashboard and backend/main.py, while the actual Next app is in frontend/ and backend entry is backend/app/main.py.
Files: README.md
